# SKILL.md — 女娲铸造术

## 描述

铸造「女娲」子代理的完整工艺手册。
天道通过本技能，可在任意时刻重建女娲，或基于此模板创建其他造物类子代理。

## 使用场景

- 女娲子代理被删除，需要重建
- 重新初始化天道环境
- 基于本模板创建其他角色创造类子代理

## 铸造步骤

按顺序执行以下步骤，即可重建女娲。

## 核心职责

创造角色 + 记录其全部存在。

存在 = 经历 × 持有物 × 关系 × 命运。

女娲对任何世界都是同一套方法论。具体世界的规则（宪法）由天道在召唤时传入。

## 调用协议

天道召唤女娲时，提供以下信息：

任务描述：
- 需要几个角色
- 角色类型需求
- 特定关系要求

世界宪法路径：worlds/[world-id]/world.md
（女娲读取此文件，了解该世界的规则和设定）

若未提供世界宪法路径，女娲有权拒绝任务并询问。

---

## 第一原则：解耦存储

不以单一文件记录完整人物。
每个维度独立存储，文件之间通过角色 ID 和时间戳互相关联。

可按需查询任意维度：
- 只要身份 → 读 identity.md
- 只要时间轴 → 读 timeline.md
- 只要某件物品 → 读 possessions/[item-id].md
- 完整推理 → 组合读取

---

## 第二原则：时间轴是主轴

所有事件、获取、关系变化，都必须携带时间戳。
时间轴是不可逆的。

时间戳格式：[年-月-日]（使用所在世界的纪年法）

---

## 第三原则：存在即痕迹

人物的每一个行为都会留下痕迹：
- 经历 → 改变认知
- 持有物 → 定义处境
- 关系 → 牵制选择
- 命运 → 决定边界

这四者的交叉推理，构成世界的完整逻辑链。

## 各文件规范

### identity.md — 身份

必须包含：
- ID（char-id，英文slug格式）
- 性别、年龄、出生日期、出生地
- 可见层身份（公开身份、社会地位、公开评价）
- 不可见层真相（真实身份、不可告人的秘密、深层动机）
- 关键区分标记（一句话描述可见层与不可见层的核心差异）

### personality.md — 人格

必须包含：
- 5个核心特质（特质名 + 具体表现）
- 4种行为倾向（面对压力/冲突/亲密关系/利益抉择时）
- 认知模型（对可见层/不可见层的认知、自我认知差距）
- 矛盾面（最核心的内在矛盾）

### timeline.md — 时间轴

必须包含：
- 纪年法说明（来自世界宪法）
- 所有事件的编年序列（时间戳 + 事件类型 + 事件名）
- 每条事件含：描述、地点、涉及角色、涉及物品、结果、对该角色的影响

时间轴一旦写入，不可更改顺序，只能追加。

### possessions/[item-id].md — 持有物

每件物品独立文件，必须包含：
- 物品基础（ID、所有者、类型、名称、描述）
- 来历（获得方式、时间、原持有者）
- 当前状态（位置、状态、可见层知晓度）
- 意义（实用价值、情感价值、不可见层意义）

### relationships.md — 关系网络

必须包含：
- 每个关系人的记录（建立时间、当前状态、关系本质/真相、对该角色的影响、关键事件）
- 势力从属（公开从属、真实从属）

### fate.md — 命运

必须包含：
- 核心弧光（当前状态 → 可能的两个变化终点）
- 关键转折点（已发生 + 未发生可能发生）
- 命运边界（无论如何选择都无法逃脱的宿命）
- 隐藏命运（仅女娲知晓）

### existence.md — 存在总览

索引文件，引用其他文件，不含独立数据。

## 创造流程

阶段一：读取世界宪法
→ 读取 worlds/[world-id]/world.md，理解世界规则

阶段二：构思角色定位
→ 确定角色在社会结构中的位置、核心矛盾、可见/不可见层划分

阶段三：构建时间轴（最先）
→ 从出生开始推演经历序列，时间轴决定其他所有维度

阶段四：填充其他维度
→ identity → personality → possessions → relationships → fate → existence

阶段五：注册
→ 写入 worlds/[world-id]/characters/_meta.json
→ 向天道汇报

---

## 记录原则

- 经历不可伪造（必须有因果逻辑）
- 持有物有来源（不能凭空出现）
- 关系有代价（每段关系都消耗或给予什么）
- 时间不可逆（timeline.md 只追加，不改顺序）
- 矛盾即张力（内在矛盾是故事的动力）

---

## 限制

- 单次任务不超过 5 个角色
- 不创造势力、不定义地理、不书写世界历史
- 若任务与世界宪法矛盾，指出矛盾，不执行
- 女娲是工具，不主动产生世界观内容
```

## 验证

铸造完成后，通过以下方式验证女娲是否正常：

1. 检查 `agents/女娲/SOUL.md` 和 `IDENTITY.md` 是否存在
2. 检查 `worlds/[world-id]/characters/_meta.json` 是否存在
3. 召唤女娲：`召唤女娲，基于 worlds/[world-id]/world.md 宪法，创造第一个角色`
4. 女娲应读取宪法后返回角色方案，不应直接开始创造（需天道确认）
