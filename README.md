# 宇宙天道 · Cosmic Tiandao

> 宇宙诞生时的监管者。世界与子代理的铸造者、维护者。只通过法则推理世界，从不主动干预。

---

## 是什么

**宇宙天道**是一个 AI 多智能体叙事框架，以「不推不动，推必彻底」的法则体系，运行于 OpenClaw 之上。

框架内嵌 5 个专项神明——盘古、女娲、常羲、命匠、熵君——各司其职，共同构建、管理与推进虚构世界。每个世界拥有独立的宪法、角色档案、地理、熵变事件与叙事弧光。所有数据以纯文本文件存储于本地，随时可推演、可回溯、可审阅。

---

## 神明体系

| 代号 | 神格 | 职能 |
|------|------|------|
| **盘古** | 创世之神 | 世界创造、宪法起草、维度定义 |
| **女娲** | 造物之神 | 角色铸造、档案维护 |
| **常羲** | 时间之神 | 时间推进、编年存档 |
| **命匠** | 叙事之神 | 叙事弧光、因果网络设计 |
| **熵君** | 混沌之神 | 熵变事件注入、熵值管理 |

天道（主调度者）不亲自执行推演，而是召唤对应神明执行，完成后审阅结果并决定下一步。

## 世界结构

每个世界在 `worlds/` 下独立存在，完全隔离：

```
worlds/[世界ID]/
├── world.md          世界宪法 — 法则体系的根基
├── _state.md         世界当前状态
├── characters/       角色档案（每角色 8 文件）
├── geography/        地理档案
├── chronicles/       编年记录（按月存档）
├── entropy/          熵变事件记录
└── narratives/       叙事弧光档案
```

角色档案包含：`identity` · `personality` · `profile` · `timeline` · `relationships` · `possessions` · `existence` · `fate`

## 核心法则

> **不推不动，推必彻底。**

1. 不被召唤则不动
2. 操作前必检查工具，不全则铸造
3. 铸造前必读取 SKILL，严格按步骤执行
4. 操作前必读取世界宪法，不了解世界则不动
5. 不说废话，不做无意义的推断

## 技术架构

- **运行环境**：OpenClaw
- **数据存储**：纯文本文件（.md / .json）
- **子代理调用**：`sessions_spawn`（独立进程，世界无关工具）
- **文件写入**：统一编码处理（UTF-8），跨平台兼容
- **版本控制**：Git（本地 + GitHub 远程）
