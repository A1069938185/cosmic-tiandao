# 任务摘要 — 2026-04-14T12:55

## 目标

审查 WebUI 项目，分析框架与世界解耦问题，制定重构方案。

## 审查发现

### WebUI 源码结构
- 6个源文件（types/App/main/api + 2组件 + 3页面）
- React 18 + Vite + react-router-dom + zustand + react-query
- API 客户端指向 /api 前缀

### 耦合问题（5处）
1. `types.ts` — `CharacterMeta.layer` 硬编码 `'visible'|'invisible'|'both'`
2. `types.ts` — `WorldState.entropy` 硬编码 `{ visible, invisible }` 二元结构
3. `EntropyGauge.tsx` — 只渲染两条 bar（visible + invisible）
4. `CharacterCard.tsx` — `LAYER_COLOR` 和中文标签硬编码冰山概念
5. `WorldDashboard.tsx` — overview tab 传参绑死二元熵值

### 解耦方案
核心思路：**世界维度（dimensions）从世界宪法驱动，UI 动态渲染**
- types.ts: `layer` → `layerTags: string[]`，`entropy` → `number[]`，新增 `WorldDimension` 接口
- world.md: 新增 `## 世界维度` 结构化段（YAML格式）
- EntropyGauge: 接收 dimensions[] + entropy[]，动态渲染 N 条 bar
- CharacterCard: 从 dimensions 查找 layer 信息，动态渲染 tags
- API: 解析 world.md 维度段

### 验证：新世界适配
- 冰山世界：visibility 维度，2层（可见/不可见）
- 修仙世界：realm 维度，3层（天/人/冥）
- 赛博世界：network 维度，3层（表网/深网/暗网）
- 均无需改代码，只需在 world.md 中定义维度

## 状态
方案已提出，等待用户确认后执行重构。
